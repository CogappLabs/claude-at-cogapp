## What it's bad at

- Long-running stateful tasks (loses thread)
- Mass refactors needing 100% certainty
- Visual judgement (UI polish, design taste)
- Novel-domain reasoning (no training data)
- Anything requiring real-world side-effect knowledge
- Confidently wrong about: API versions, library quirks, project conventions it hasn't been told
