## Build your own CLI, then skill it

A skill can shell out. So if a vendor has no MCP, build a small CLI yourself, then wrap it in a skill.

<TwoCol>
<div>

**Example: `cf` (Cloudflare read-only CLI)**

```text
cf zones list
cf logs http --status 503 --last 1h
cf analytics top-paths --zone example.com
cf security events --action block
cf dns records list --zone example.com
```

Built with Bun + bunli. Auth token in `~/.config/cf-cli/token.json`. Read-only by design (no destructive endpoints exposed).

</div>
<div>

<Terminal title="claude" text={cfRun} />

**Skill wrapping it:**

```md
---
name: cloudflare-cli
description: >
  Query Cloudflare analytics, logs, security
  events, DNS via the cf CLI. Use for traffic
  questions, bot checks, WAF events.
---

Use `cf` (at ~/git/cloudflare-cli).
- `cf logs http` for HTTP requests
- `cf security events` for WAF
- `cf analytics top-paths` for traffic
```

</div>
</TwoCol>

CLI does the talking to the API. Skill teaches Claude when + how to call it. Cheap to add new vendors.
