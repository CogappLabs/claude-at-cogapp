## Status line + hooks

Bottom-of-terminal info bar. Shell command, runs each turn. JSON in via stdin, plain text out.

<TwoCol>
<div>

`~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "/bin/bash ~/.claude/statusline.sh"
  }
}
```

`statusline.sh` (excerpt):

```bash
input=$(cat)
model=$(echo "$input" | jq -r '.model.display_name')
ctx=$(echo "$input" | jq -r '.context_window.used_percentage')
cost_calc_from_tokens_and_model_pricing
git_branch=$(git symbolic-ref --short HEAD)

printf "%s 🦄 Ctx: %d%% \$%.3f %s" \
  "$model" "$ctx" "$cost" "$git_branch"
```

</div>
<div>

<Terminal title="status line" text={statusOut} />

**What it shows**

- Model + context use %
- 5-hour rate-limit % + reset countdown
- Session cost ($)
- Git branch + dirty marker

Hooks run on other events too: `Stop` (worklog), `SessionStart`, `UserPromptSubmit`, pre/post tool calls.
